QT4 XPath 4 equivalent functions

These are the available XSLT 3 functions that approximately emulate new or modified XPath 4 functions. The links point to the specification definition:

  • Functions marked  ✚ are new in XPath 4.0; those marked  ✭ are modified.
  • The namespace prefix mappings xmlns:f="MyFunctions" xmlns:ar="MyArray" xmlns:ma="MyMap" should already be defined in the example stylesheets
  • Calls can only be made by positional arguments - keyword binding is not supported.
  • In general the function-valued arguments must have the maximum arity given in the specification, i.e. optional arguments of the inline function must be present. (SaxonJS will complain of arity mismatch)
  • These functions are generally type-permissive on function-valued arguments,
    e.g. where the specification would restrict an argument to function(item()*,xs:integer),
    the emulation only restricts to function(*).
    Thus run-time failures may occur which in a true implementation could be caught at compile-time.
